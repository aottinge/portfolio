Projet 3 : Site de E-commerce en Flask

Analyse : Ce projet consistait à développer un site de E-commerce complet avec Flask. J'ai mis en place la gestion des produits, du panier, des commandes, d'un historique, d'une liste d'envie et de l'authentification des utilisateurs.

Ce projet m'a permis de consolider mes compétences en développement et de comprendre les enjeux de la sécurité et de la gestion des données dans une application réelle.

Trace : La Figure 1 est une capture de l’environnement de développement Visual Studio Code, illustrant une partie du code Python d’un projet web basé sur Flask. Le fichier affiché (client_liste_envies.py) contient une fonction appelée client_liste_envies_show, associée à une route /client/envies/show. Cette fonction gère l’affichage des données relatives aux envies et à l’historique de consultation d’un client connecté.

Le code met en œuvre des requêtes SQL exécutées directement via un curseur obtenu à partir d’une base de données SQLite ou MySQL (fonction get_db().cursor()). Plusieurs traitements sont réalisés successivement :

  • D’abord, la récupération de la liste des envies du client, triée par date d’ajout, en joignant trois tables (liste_envie, velo, type_velo).
  • Ensuite, la récupération de l’historique de consultation des articles (ici des vélos), limité aux 6 derniers articles consultés, en triant par date de consultation.
  • Enfin, deux requêtes supplémentaires permettent de compter le nombre total d’articles dans la liste d’envies et dans l’historique.

Cette figure permet de mettre en évidence des savoirs liés à la programmation web en Python avec Flask, à la manipulation de bases de données relationnelles, et à l’écriture de requêtes SQL avec jointures. On observe également des savoir-faire tels que le traitement sécurisé des requêtes SQL avec paramètres, la gestion de sessions utilisateur, et la capacité à structurer un traitement côté serveur pour préparer les données destinées à l’affichage dans une interface web.

Le code est lisible, bien commenté, et montre une bonne maîtrise des étapes de collecte et d’organisation des données à partir d’une base, en réponse à une requête HTTP. Il s’inscrit donc dans une logique typique de développement d’une application web dynamique.

Trace : La Figure 2 est une capture d'écran d'une interface web de visualisation statistique, réalisée dans le cadre d'un projet de développement en Python, en lien avec une base de données e-commerce. Elle illustre l'exploitation de la datavisualisation (dataviz) pour représenter et analyser des données issues du comportement des utilisateurs : ici, les articles ajoutés aux wishlists (listes d'envies) et les consultations par catégorie de vélos.

Dans la partie supérieure de l'interface, un tableau synthétise les données par catégorie : nombre d'articles présents dans les wishlists et nombre de consultations sur les 30 derniers jours. Ces chiffres sont automatiquement extraits et agrégés depuis la base, via des requêtes SQL. En dessous, deux graphiques sont générés pour offrir une lecture plus visuelle des données : un histogramme pour les articles dans les wishlists, et un autre pour les consultations en fonction de la catégorie.

Cette figure met en évidence des savoirs liés à la représentation graphique de données avec une bibliothèque de dataviz et Flask. Elle mobilise également des savoir-faire techniques comme la connexion dynamique entre les requêtes SQL et les éléments graphiques, le rafraîchissement conditionnel des données, ou encore l'affichage interactif piloté par des menus déroulants.

Enfin, cette interface est un bon exemple de traitement et visualisation de données en temps réel, utile pour l'analyse d'activité utilisateur ou la prise de décision marketing. Elle prouve la capacité à transformer des données brutes en indicateurs clairs, adaptés à une interface web moderne.
